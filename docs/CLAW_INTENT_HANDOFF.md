# Claw 意图交接

这份文档说明 DARWIN 对外所说的 `Claw 驱动` 在产品层面具体是什么意思。

## 用户意图

示例自然语言目标：

> 给我一个保守、以本地运行优先的 BTC / ETH 交易姿态；如果还是震荡，就优先做区间策略；只有在趋势被确认后才扩大方向风险。

## Claw 归一化后的操作目标

Claw 负责把上面的自然语言目标归一成 DARWIN 可执行的操作目标：

- 风控等级：`balanced`
- 白名单资产：`BTC-USDT, ETH-USDT, SOL-USDT`
- 优先姿态：震荡优先，趋势确认后再提升方向暴露
- 治理要求：
  - 所有策略先通过 shadow-first 生命周期
  - 风控通过 4 层熔断执行
  - 执行和审计保持在同一条责任链上

## DARWIN 如何接手

DARWIN 不负责“理解口语目标”，它负责把归一化后的操作目标落到系统里：

1. `Overview`
确认权益、市场主导状态、策略总量、系统健康。

2. `Decision`
解释为什么当前市场状态对应这组策略。

3. `Risk`
确认 4 层熔断和审批门禁是否生效。

4. `Reports`
生成日报与审计记录，证明所有动作都留在责任链上。

## 零密钥演示入口

```bash
pnpm run proof
```

这个命令会在本地输出一份确定性的 `Claw -> DARWIN` 交接演示，不需要交易所密钥。
