# DARWIN 意图管线

这份文档说明自然语言目标如何进入 DARWIN，并被整理成可执行的系统目标。

## 用户目标

示例自然语言目标：

> 给我一个保守、以本地运行优先的 BTC / ETH 交易姿态；如果还是震荡，就优先做区间策略；只有在趋势被确认后才扩大方向风险。

## 结构化后的操作目标

DARWIN 使用结构化操作目标来驱动后续市场识别、策略选择和风险控制：

- 风控等级：`balanced`
- 白名单资产：`BTC-USDT, ETH-USDT, SOL-USDT`
- 优先姿态：震荡优先，趋势确认后再提升方向暴露
- 治理要求：
  - 所有策略先通过 shadow-first 生命周期
  - 风控通过 4 层熔断执行
  - 执行和审计保持在同一条责任链上

## 系统接收后会做什么

1. `Overview`
确认权益、市场主导状态、策略总量和系统健康。

2. `Decision`
根据当前市场状态解释策略选择和资金分配。

3. `Risk`
确认 4 层熔断和审批门禁是否生效。

4. `Reports`
生成日报与审计记录，保留完整运行轨迹。

## 本地查看入口

```bash
pnpm run overview
```

这个命令会在本地输出一份确定性的系统概览，不需要交易所密钥。
